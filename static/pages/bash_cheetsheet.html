Bash cheet sheet

Navigation
pwd Print absolute path to working (current) directory.
ls List directory contents in columns.
ls -l As table, with more details.
ls -a List all files, including hidden ones (ones starting with . (dot)).
cd Change working directory.
cd [path] Change working directory to [path]. It can be both absolute and relative, ~ is used if not specified. [path] usage:
  1. a b >> folder 'a b' exists, does not work
  2. 'a b' >> works because of '' usage
  3. a\ b >> works because ' ' (space) symbols are used
Ctrl+C Stop process: sends signal SIGINT to stop terminal process.
Ctrl+Z Put current process to background.
man (help)
man Format and display the on-line manual pages (does not work this way).
man [command] Display man help page for [command]
man -k [term] Do a keyword search for all manual pages containing the given search term.
/[term] Within a manual page, perform a search for [term]
n After performing a search within a manual page, select the next found item.
Working with files and folders
touch [path] Create empty file at [path]
mkdir Makes (creates) directories.
mkdir [path] Create directory at [path], both absolute and relative. Multiple folders can be created using ' ' (space) as separator. Only creates folder if parent folders are created.
mkdir -p [path] Creates required parent directories if they are required during path creation, recursively.
mkdir -v [path] Tell us what it does, prints a message for each created directory. Works by typing --verbose as well.
cp [source] [destination] Copy [source] to [destination], does not create destination folder.
cp -r [source] [destination] Copy [source] to [destination], can create one folder in depth
mv [path1] [path2] Moves and/or renames [path1] to [path2]
rm [path] Remove a file located at [path]
rm -r [path] Remove not empty folder located at [path]
rmdir [path] Remove empty directory at [path]
rmdir -r [path] Remove non-empty directory at [path]
vi editor usage
vi [path] Edit file located at [path] in vi editor
:ZZ Capitals! Save and exit
:q! discard all changes since the last save, and exit
:w save file but don't exit
:wq save and exit
Opening files
more [path] Print file at [path] line per line.
tac [path] Print file at [path] line per line reversed.
less [path] Open file at [path] in less, cannot be edited.
nano [path] Open file at [path] in nano editor.
head -[lines] [path] print [lines] (number) top lines from file at [path]
tail -[lines] [path] print [lines] (number) bottom lines from file at [path]
Permissions
chown change file owner and group
[drwx] Letters used in permissions:
  • d — directory
  • r — read
  • w — write
  • x — execute
Example: [-/d][---/rwx][---/rwx][---/rwx]

File type (file/directory) /// Owner rights /// Group rights /// Everyone rights
[777] [rwx][rwx][rwx] — binary representation, - is 0 while r/w/x is 1: rwx is 7, r--r--rwx is 447
Working with files
sort [options] [path] Sort lines of file located at [path] and print them, [options] can customize it.
nl [path] Print file at [path] with line numbers.
wc [path] Word count: lines, words and characters.
wc -lwc [path] Word count: (L)ines, (W)ords and (C)haracters. Remove letters from lwc if one of them are not required.
sed 's/[from]/[to]/g' file Search [from] term and exchange to [to], does not write to file.
uniq [path] Remove duplicate lines from file at [path]
Regular expressions (RegExp)
egrep [regexp] [path] Search [path] for [regexp] and print matching lines.
egrep -n [regexp] [path] Search [path] for [regexp] and print matching lines with their line numbers.
egrep -c [regexp] [path] Search [path] for [regexp] and print how many matching lines were found.
Regular expressions (writing of)
\d (digit) Most engines: one digit, 0-9 file_\d\d file_25
.NET, Python3: one Unicode digit in any script, 0-9 file_\d\d file_9੩
\w (word character) Most engines: ASCII letter, digit or underscore \w-\w\w\w A-b_1
Python3: Unicode letter, ideogram, digit, or underscore \w-\w\w\w 字-ま_۳
Unicode letter, ideogram, digit, or connector \w-\w\w\w 字-ま‿۳
\s (whitespace character) Most engines: space, tab, newline, carriage return, vertical tab a\sb\sc a b c
.NET, Python 3, JavaScript: any Unicode separator a\sb\sc a b c
\D (not digit) One character that is not a digit as defined by your engine's \d \D\D\D ABC
\W (not word) One character that is not a word character as defined by your engine's \w \W\W\W *=)
\S (not whitespace) One character that is not a whitespace character as defined by your engine's \s \S\S\S\S Yoyo
+ One or more Version \w-\w+ Version A-b1_1
{3} Exactly three times \D{3} ABC
{2,4} Two to four times \d{2,4} 156
{3,} Three or more times \w{3,} regex_tutorial
* Zero or more times A*B*C* AAACC
? Once or none plurals? plural
. Any character except line break a.c abc
.* whatever, man.
\. A period (special character: needs to be escaped by a \) a\.c a.c
\ Escapes a special character \.\*\+\? \$\^\/\\ .*+? $^/\
\[\{\(\)\}\] [{()}]
| Alternation, OR operand, logical OR 22|33 33
() Capturing group A(nt|pple) Apple (captures "pple")
^ Beginning of the line
$ End of line

You can check more details about regular expressions at rexegg.com

Pyping commands
ls > [path] Save output as file at [path]
ls >> [path] Add output to end of file at [path]
ls 2> [path] Save errors to file at [path]
ls | head -3 Send output of the left command to right one as input.
top Shell analogue of Task Manager which displays processes.
ps aux Report snapshot of all the current processes:
  1. a — Select all processes except both session leaders(?) and processes not associated with a terminal.
  2. u - Select by effective user ID (EUID) or name. This selects the processes whose effective user name or ID is in userlist. The effective user ID describes the user whose file access permissions are used by the process. Identical to U and --user.
  3. x - Lift the BSD-style "must have a tty" restriction, which is imposed upon the set of all processes when some BSD-style (without "-") options are used or when the ps personality setting is BSD-like. The set of processes selected in this manner is in addition to the set of processes selected by other means. An alternate description is that this option causes ps to list all processes owned by you (same EUID as ps), or to list all processes when used together with the a option.
  4. w - Wide output. Use this option twice for unlimited width.
ps auxw | grep [term] Output all processes matching [term]
kill [PID] Kill process by [PID] (process id).
kill -9 6978 Kill process for sure.
sleep [time] Wait [time] seconds. Can be indicated as 5s/m/h/d to refer to seconds/minutes/hours/days
which [command] Returns [command] location.
